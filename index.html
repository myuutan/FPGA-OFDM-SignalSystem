FPGA Implementation of a Railway Signaling System Using the OFDM Transmission Method

Ryo Higuchi¹, Hiroshi Mochizuki²
¹Graduate School, College of Science and Technology, Nihon University
²Faculty, College of Science and Technology, Nihon University

Abstract: Currently, railway signaling systems that transmit control information via rails, such as the Automatic Train Control (ATC) systems in Japan, are widely used. However, increasing the transmission speed is difficult due to the transmission characteristics of the rails. To overcome this problem, we have proposed a new transmission method using Orthogonal Frequency Division Multiplexing (OFDM). We previously determined a detailed data allocation scheme using majority decision to improve the Bit Error Rate (BER) characteristics. In this paper, we describe the FPGA implementation of our proposed method and evaluate the performance of the implemented device.

1. Introduction
Currently, railway signaling systems using track circuits (rails) as the transmission medium, represented by systems like Automatic Train Control (ATC), are widely employed. While using track circuits as a medium offers advantages in safety, such as being less susceptible to interference compared to wireless methods, it also presents challenges. These include strong influence from unique rail noise generated by trains and the inability to achieve high transmission speeds due to the use of the audible frequency band.
To address this issue, we have been investigating a method to introduce Orthogonal Frequency Division Multiplexing (OFDM) transmission into track circuits. OFDM allows for the superposition of many carriers within a limited frequency band by narrowing the spacing between subcarriers [1]. Previous research has identified a data allocation method to improve the Bit Error Rate (BER) characteristics for realizing an OFDM-based railway signaling system [2].
Building on this, our current study focuses on the implementation of this system using a Field-Programmable Gate Array (FPGA). Specifically, we evaluate the implementation of the Fast Fourier Transform (FFT) and Inverse FFT (IFFT), which are key components of OFDM transmission. We also construct an OFDM signal generation unit and evaluate its output characteristics.

2. Characteristics of Transmission Using Track Circuits
Track circuits exhibit significant attenuation at high frequencies above 10 kHz. Consequently, it is not possible to use frequency bands like those in general wireless communications, restricting transmission to the audible frequency band.
Regarding noise, the system is affected by rail noise that differs from typical white noise and contains numerous harmonic components of the commercial power frequency from the return current. Figure 1 shows an example of the spectral distribution of rail noise. From this figure, it is evident that noise with peaks at multiples of 300 Hz (the 6th harmonic of the commercial power frequency) is dominant. Another characteristic is that this noise fluctuates over time.

3. Railway Signaling System Using OFDM Transmission
Based on the track circuit transmission characteristics described in the previous section, a block diagram of the proposed railway signaling system using OFDM is shown in Figure 2. OFDM is a multicarrier method that uses a large number of carriers. Even if the transmission bands of the subcarriers overlap, they do not interfere with each other due to their orthogonal relationship. This allows for high spectral efficiency, which is considered effective for narrowband transmission paths.
Furthermore, against rail noise that has peaks at specific frequencies, it is possible to improve the Bit Error Rate (BER) by allocating the same data to multiple carriers.

4. Investigation and Evaluation of FPGA Implementation
Based on our previous results, we proceeded with the actual FPGA implementation. The computer simulations in this study were conducted using MATLAB/Simulink. We leveraged this model to generate HDL (Hardware Description Language), which then served as the basis for the FPGA implementation.
Figure 5 shows a comparison of the outputs from the MATLAB/Simulink simulation model and the model implemented on the FPGA. Specifically, when data is applied to a single subcarrier, a positive output is generated in the imaginary part of the corresponding subcarrier, and a negative output is generated in the conjugate location. As Figure 5 shows, we confirmed that the FPGA-implemented model, even after modifications such as fixed-point conversion, produced an output similar to the MATLAB/Simulink model.
Furthermore, we implemented an OFDM transmitter on the FPGA that applies data to 219 subcarriers and confirmed that an OFDM signal spectrum, as shown in Figure 6, could be obtained.

5. Conclusion
In this study, we evaluated the performance of our proposed railway signaling system using OFDM transmission when implemented on an FPGA. For future work, we plan to implement the receiver unit and also conduct performance evaluations through actual field tests, with the aim of further advancing this research.

FPGA Implementation for Railway Signaling System Using OFDM Transmission method

Introduction
Currently, railway signaling systems using track circuits (rails) represented by ATC (Automatic Train Control) systems are widely used. While using track circuits as a transmission medium has advantages in terms of safety, such as easier consideration of interference waves compared to wireless methods, there are also problems such as strong influence of railway noise peculiar to trains and the inability to increase transmission speed due to transmission in the audible frequency band. To address this problem, we are investigating a method of introducing OFDM (Orthogonal Frequency Division Multiplexing) transmission, which can superimpose many carriers within a limited frequency band by narrowing the spacing between subcarriers, into track circuits. In previous research, we clarified a data assignment method to improve Bit Error Rate (BER) characteristics when implementing a railway signaling system applying OFDM transmission.

Based on this, in this study, we investigated actual implementation using FPGA (Field Programmable Gate Array), which is a programmable device. Specifically, we evaluated the implementation of FFT (Fast Fourier Transform) and IFFT (Inverse FFT), which are key components of OFDM transmission, and also constructed an OFDM signal generation unit and evaluated its output characteristics.

Characteristics of Transmission Using Track Circuits
Track circuits have characteristics of large attenuation at high frequencies of 10kHz or more, so general radio communication frequency bands cannot be used, and transmission is carried out in the audible frequency band. Regarding noise, unlike general white noise, it is noise containing many harmonic components of commercial frequencies included in the return current. Figure 1 shows an example of the spectral distribution of railway noise. From this figure, it can be seen that the 6th harmonic of the commercial frequency, i.e., 300Hz, has a peak. It is also characterized by temporal fluctuations.

Railway Signaling System Using OFDM Transmission
Figure 2 shows a block diagram of a railway signaling system using OFDM transmission, based on the characteristics of track circuit transmission shown in the previous section. OFDM transmission is one of the multi-carrier methods that uses multiple carriers, and even if the transmission bands between subcarriers overlap, they do not interfere due to their orthogonal relationship. Therefore, it is considered effective in narrow-band transmission paths as it can improve frequency utilization efficiency. Furthermore, for railway noise that has peaks at specific frequencies, there is a possibility of improving the bit error rate (BER) characteristics by placing the same data on multiple carriers. Figure 3 shows the BER characteristics when computer simulations were performed for each subcarrier when railway noise and white noise were added. From this figure, it was confirmed that while errors occurred uniformly across each subcarrier with white noise, with railway noise, bit errors were concentrated on specific subcarriers. Furthermore, since commercial frequencies are expected to fluctuate in practice, considering this, as shown in Figure 4, by assigning the same data to subcarriers with different intervals from the frequency intervals of harmonic components, a configuration was adopted to prevent errors even when railway noise frequency fluctuates, and it was confirmed through computer simulation that the BER characteristics could be improved compared to conventional assignments.

Examination and Evaluation of FPGA Implementation
Based on the results so far, we decided to proceed with FPGA implementation. Here, in this computer simulation, we conducted the study using MATLAB/Simulink, so we decided to generate HDL (Hardware Description Language) based on this model and then implement it on FPGA. Figure 5 shows a comparison of the output between the MATLAB/Simulink computer simulation model and the model implemented on FPGA. Specifically, when data is given to one subcarrier, a positive output of the imaginary part is produced for the corresponding subcarrier, and a negative output of the imaginary part is produced at a conjugate position. Considering this, looking at Figure 5, it was confirmed that the FPGA implementation model, even after changes such as fixed-point conversion, could obtain the same output as the MATLAB/Simulink model. Furthermore, an OFDM transmitter with 219 subcarriers was implemented on FPGA, and it was also confirmed that an OFDM signal spectrum as shown in Figure 6 could be obtained.

Conclusion
This time, we evaluated the performance of our proposed railway signaling system using OFDM transmission when actually implemented on FPGA. In the future, we plan to implement the receiver section on FPGA and also conduct performance evaluations through actual field tests to further deepen our research.
